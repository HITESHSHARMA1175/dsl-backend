# DSL Backend (Node.js)

REST API backend for the DSL Clinic platform — a full conversion of the original Laravel application to **Express.js + TypeScript + Prisma (MySQL)**.

## Tech stack
- Node.js + Express.js (TypeScript)
- Prisma ORM (MySQL)
- JWT authentication (admin, customer, agent)
- Zod request validation
- Swagger / OpenAPI docs

## Getting started

```bash
# 1. Install dependencies
npm install

# 2. Configure environment
cp .env.example .env
# then fill in DATABASE_URL, JWT secrets, Stripe/Klarna keys, SendGrid key

# 3. Sync Prisma client
npx prisma generate

# 4. Run in development
npm run dev
```

- Server: http://localhost:3000
- API base: http://localhost:3000/api/v1
- Swagger docs: http://localhost:3000/api-docs

## Deploying to Render

This repo includes a `render.yaml` blueprint.

1. Push the repo to GitHub (already done).
2. In Render: **New → Blueprint** → connect this repo → Render reads `render.yaml`.
   - Or **New → Web Service** with:
     - Build command: `npm install`
     - Start command: `npm start`
     - Health check path: `/health`
3. Set environment variables in the Render dashboard:
   - `DATABASE_URL` — a **hosted** MySQL connection string (Render does not host MySQL; use Railway, PlanetScale, Aiven, Clever Cloud, etc.). Format: `mysql://user:pass@host:3306/dbname`
   - `JWT_ACCESS_SECRET`, `JWT_REFRESH_SECRET` — auto-generated by the blueprint.
   - Optional: `STRIPE_SECRET_KEY`, `SENDGRID_API_KEY`, Klarna creds.
4. Deploy. Render gives a URL like `https://dsl-backend.onrender.com`.

Once live:
- **Swagger UI:** `https://<your-app>.onrender.com/api-docs`
- **OpenAPI JSON:** `https://<your-app>.onrender.com/api-docs.json`
- **API base for Postman:** `https://<your-app>.onrender.com/api/v1`

Note: the app boots and serves Swagger even before `DATABASE_URL` is set; data
endpoints return errors until a reachable MySQL is configured.
